Coupled Oscillators

Two masses connected by springs: a classic linear system.

Newton's second law for each mass:

$m_1 \ddot{x}_1 = -k_1 x_1 + k_2(x_2 - x_1)$
$m_2 \ddot{x}_2 = -k_2(x_2 - x_1) - k_3 x_2$
$\frac{d\mathbf{x}}{dt} = A\,\mathbf{x}$
All coupled linear systems have this form. The matrix $A$ encodes all the physics.

When Is a System Linear?

Linear: $A$ is constant

$\dot{\mathbf{x}} = A\,\mathbf{x}$

Coupled oscillators, circuits,
small perturbations, heat equation
Nonlinear: $A$ depends on state

$\dot{\mathbf{x}} = f(\mathbf{x})$

Pendulum (large angle), Lorenz,
Navier-Stokes, population dynamics
Even nonlinear systems are locally linear (linearization around fixed points). DMD exploits this idea globally.

Eigendecomposition of $A$

Recall from last lecture: any square matrix can be decomposed as

$A = V \Lambda V^{-1}$
$V$ = eigenvectors (columns)
Mode shapes: how components move together
$\Lambda$ = diagonal eigenvalues
Growth rates: how fast each mode evolves
For symmetric $A$: all $\lambda_i$ are real, $V$ is orthogonal.

The Decoupling Trick

Change coordinates: $\mathbf{z} = V^{-1}\mathbf{x}$

$\frac{d\mathbf{z}}{dt} = \Lambda\,\mathbf{z}$

Each component is independent:

$\dot{z}_i = \lambda_i\, z_i \quad\Longrightarrow\quad z_i(t) = z_i(0)\,e^{\lambda_i t}$
Eigenvalues determine everything: growth, decay, and oscillation are all encoded in $\lambda_i$.

What Do Eigenvalues Tell Us?

For $z_i(t) = z_i(0)\,e^{\lambda_i t}$, the eigenvalue $\lambda_i$ controls the behavior:

$\lambda > 0$ (real)

Exponential growth
$\lambda < 0$ (real)

Exponential decay
$\lambda = \sigma \pm i\omega$

Oscillation (± growth/decay)
$e^{(\sigma + i\omega)t} = e^{\sigma t}\bigl(\cos\omega t + i\sin\omega t\bigr)$

Back to $\mathbf{x}$: Superposition of Modes

The full solution in original coordinates:

$\mathbf{x}(t) = \sum_{i=1}^n c_i\, e^{\lambda_i t}\, \mathbf{v}_i$
Each mode $\mathbf{v}_i$ evolves independently at rate $\lambda_i$.
The motion is their superposition.
But what if we don't know $A$? What if we only have data?

From Models to Data

In the real world, we measure snapshots $\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_m$ at successive times.

$X = \begin{bmatrix} | & | & & | \\ \mathbf{x}_1 & \mathbf{x}_2 & \cdots & \mathbf{x}_{m-1} \\ | & | & & | \end{bmatrix}$

Shifted matrix:

$X' = \begin{bmatrix} | & | & & | \\ \mathbf{x}_2 & \mathbf{x}_3 & \cdots & \mathbf{x}_{m} \\ | & | & & | \end{bmatrix}$
If the system is linear:

$X' \approx A\, X$

Each snapshot maps to the next via $A$.

Solving for $A$

From $X' = AX$, we can solve for $A$ using the pseudo-inverse:

$A = X'\, X^\dagger = X'\, V\, \Sigma^{-1}\, U^T$

Equivalently, this is a least-squares problem:

$A = \arg\min_A \|X' - AX\|_F^2$
Problem: If $\mathbf{x}$ has $n$ components (e.g., 89,351 pixels), then $A$ is $n \times n$.
That's an 8 billion entry matrix! We need a trick.

The SVD Projection Trick

Start from the SVD of $X$ (recall Lecture 9):

$X = U\Sigma V^T \quad\longrightarrow\quad X \approx U_r \Sigma_r V_r^T$

Project $A$ onto the $r$-dimensional SVD basis:

$\tilde{A} = U_r^T\, X'\, V_r\, \Sigma_r^{-1}$
$\tilde{A}$ is $r \times r$ instead of $n \times n$. For cylinder flow: $21 \times 21$ instead of $89{,}351 \times 89{,}351$.

DMD: The 4-Step Recipe

Step 1: SVD of $X$:   $X = U\Sigma V^T$   (truncate to rank $r$)
Step 2: Projected dynamics:   $\tilde{A} = U_r^T\, X'\, V_r\, \Sigma_r^{-1}$
Step 3: Eigendecomposition:   $\tilde{A}\, W = W\, \Lambda$
Step 4: DMD modes:   $\Phi = X'\, V_r\, \Sigma_r^{-1}\, W$
Cost: SVD of $X$ is $O(nm\min(n,m))$ + eigendecomp of $r\times r$ is $O(r^3)$. We never build the full $n \times n$ matrix $A$.

DMD in Python


def DMD(X, Xprime, r):
    # Step 1: SVD of X (truncate to rank r)
    U, S, Vt = np.linalg.svd(X, full_matrices=False)
    Ur, Sr, Vtr = U[:,:r], np.diag(S[:r]), Vt[:r,:]

    # Step 2: Projected dynamics
    Atilde = Ur.T @ Xprime @ Vtr.T @ np.linalg.inv(Sr)

    # Step 3: Eigendecomposition
    Lambda, W = np.linalg.eig(Atilde)

    # Step 4: DMD modes
    Phi = Xprime @ Vtr.T @ np.linalg.inv(Sr) @ W
    return Phi, Lambda
Four lines of linear algebra. The same SVD you learned last lecture does all the heavy lifting.

Interpreting DMD Output

DMD Modes $\boldsymbol{\phi}_i$

Spatial patterns (like eigenfaces,
but for dynamics).

Each mode captures a coherent
spatial structure that oscillates
at a single frequency.
Eigenvalues $\lambda_i$

Encode frequency and growth rate.

$|\lambda_i| < 1$: decaying mode
$|\lambda_i| = 1$: neutral (persistent)
$\angle\lambda_i$: oscillation frequency
Reconstruction:   $\mathbf{x}(t) = \displaystyle\sum_{i=1}^r b_i\, \lambda_i^t\, \boldsymbol{\phi}_i$

Summary

Linear Dynamical Systems: $\dot{\mathbf{x}} = A\mathbf{x}$
Eigenvalues of $A$ control growth, decay, and oscillation. Each mode evolves independently.
DMD Algorithm: Given snapshots, find $A$ efficiently via SVD projection.
4 steps: SVD $\to$ project $\to$ eigendecompose $\to$ recover modes.
Result: Spatial modes $\boldsymbol{\phi}_i$ + frequencies $\omega_i$ + growth rates $\sigma_i$.
Data-driven decomposition of complex dynamics into interpretable components.

Ref: Schmid (2010), Brunton & Kutz (2019) Ch. 7
